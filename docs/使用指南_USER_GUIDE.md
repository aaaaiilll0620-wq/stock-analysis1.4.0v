# 台股四維度決策系統 — 使用指南

> 這是一套**研究/篩選輔助工具**,幫你用一致、可回測的邏輯替個股評級與排序。
> **它不是投資建議**,也不會幫你下單。評級是對股票「當前狀態」的分類,不是「立刻買進」的指令。

---

## 1. 這套能做什麼

- **個股分析**:輸入一檔代號,看它在四大維度 (基本面/估值/技術面·動能·籌碼) 的評分、最終評級、以及買點提示。
- **多檔排行**:一次評估多檔,依綜合分排序,快速看哪些強、哪些弱。
- **綜合分選股**:讀一份事先算好的分數快照,把**整份自選名單**一次拿來跨股排名 —— 0 API、免 token、瞬間出結果,適合大範圍海選。
- **滾動回測**:用嚴格的 point-in-time (無未來函數) 驗證這套評級/排序在歷史上有沒有效。

三種使用方式:命令列 (`main.py`)、網頁 (`app.py`,Streamlit)、以及回測 (`tests/run_backtest.py`)。

## 2. 快速開始

1. 準備 FinMind Token(見第 3 節)。本機自用可放進根目錄 `.env`:`FINMIND_TOKEN=你的token`;網頁公開站則在左側欄貼自己的 token。
2. 安裝套件:`pip install -r requirements`(或至少 pandas、openpyxl;網頁再加 `streamlit`;快取再加 `pyarrow duckdb`)。
3. 跑跑看:
   - 個股 (命令列):`python main.py 2330`
   - 個股 (網頁):`pip install streamlit` 後 `streamlit run app.py`
   - 排行:`python main.py 2330 2454 2317`
   - 回測:`python tests\run_backtest.py --cache`

---

## 3. 取得 FinMind API Token

「個股分析」與「多檔排行」會即時向 **FinMind**(台股開源金融資料 API)抓最新資料,需要一組**你自己的** token。免費、用 Email 註冊就有,一分鐘內可拿到:

1. 開啟官網 👉 **https://finmindtrade.com/**
2. 點右上角 **登入 / 註冊**,用 Email + 密碼註冊,到信箱點驗證信完成啟用。
3. 登入後進入**會員中心** 👉 **https://finmindtrade.com/analysis/#/account/user**
4. 找到 **API Token(api_token 金鑰)**,整串複製起來。
5. 貼到你要用的地方:
   - **本機自用**:寫進專案根目錄 `.env` 的 `FINMIND_TOKEN=…`。
   - **網頁公開站**:貼到左側欄「你的 FinMind API token」輸入框(只存在你這次的瀏覽階段、算你自己的額度、不共用、不寫入伺服器)。

**額度(大致)**:不貼 token 的匿名模式每小時額度極低、即時查詢常失敗;**免費註冊會員每小時約 600 次**,個人研究綽綽有餘;贊助 / Sponsor 方案上限更高。詳見官網,登入說明:https://finmind.github.io/login/。

## 4. 每檔用多少 API?一次查幾檔剛好?

每分析一檔**本機還沒快取的**全新股票,大約向 FinMind 送出 **9 支請求**(股價、法人買賣超、流通股數、融資融券、月營收、本益比河流圖、損益表、資產負債表、現金流量表)。

但系統有**本機快取**:同一檔股票**當天再查幾乎不再耗 API**(直接讀快取),只有第一次抓、或加了 `--refresh` /勾了網頁「刷新最新資料」時才會實際打 API。所以真正吃額度的是「這一小時內第一次分析的股票檔數」。

以免費會員每小時約 600 次估算,600 ÷ 9 ≈ **一小時可全新分析約 60 檔**。建議:

- **個股分析**:一次 1～5 檔最順手。
- **多檔排行**:逐檔即時打 API,建議**一次 20 檔以內**、盡量不超過 **30 檔**(30 檔全新 ≈ 270 次)。
- **想一次掃上百檔** → 用「綜合分選股」(第 8 節):0 API、免 token、瞬間出。

若短時間連續分析大量沒查過的新股票而抓不到資料,通常是撞到每小時上限,**等一小時額度重置**即可,或先用綜合分選股頂著。

## 5. 三種模式怎麼選(最重要)

模式 (保守/平衡/積極) **不設定持有天數**,它決定的是「分數由哪些訊號主導、門檻多嚴」。因為動能/籌碼訊號衰退得快 (短線)、基本面/估值走得慢 (長線),所以挑**跟你實際打算持有多久相符**的模式:

| 你的情況 | 模式 | 說明 |
|---|---|---|
| 新手 / 不確定 / 中線波段 (數週~數月) | **平衡** (預設) | 以基本面+技術為核心,最穩、最不被單一雜訊帶走 |
| 長線 / 存股 (數季以上) | **保守** | 重基本面品質與估值、門檻高,選得少但扎實,抱得住 |
| 短線 (數天~數週,會主動停損) | **積極** | 重動能與籌碼點火、門檻低,抓正在噴的股;雜訊與交易成本最高 |

> 誠實提醒:回測顯示「積極」那套動能主導的權重,樣本外排序力最弱 (動能因子的 Rank IC 是負的),
> 而且短線是最難、成本最高的玩法。**多數人(包含很多自以為做短線的人)用「平衡」會更好**;
> 「積極」比較適合知道自己在做什麼、會嚴設停損的人。

命令列切換:`python main.py 2330 --mode conservative`(或 `-m c/b/a`)。網頁在左側直接點選。

## 6. 四個評級是什麼意思(常被誤解)

系統有**兩條並行的買進軌道**,不是同一件事的強弱:

- **強勢買進 = 順勢動能軌**:正在領漲的主流股 —— 均線多頭排列、動能強、法人主力狂吸,
  這時系統會**特赦**「太貴/RSI 高/乖離大」的反對票,因為成長跟籌碼撐得住。屬於「騎強勢的馬」。
- **強烈推薦 = 價值/品質軌**:便宜或合理的好公司 —— 估值偏低、基本面扎實、沒過熱、沒追高。
  屬於「用好價格買好公司」,常是回檔/打底時的進場。
- **觀望追蹤**:訊號不足或有疑慮 (例如獲利品質存疑、籌碼不足),先追蹤、不急。
- **謹慎避開**:基本面或技術面不佳,避開。

兩個重點:

1. **「強勢買進」不代表現在就要買。** 評級是對「當前狀態」的分類。很多強勢買進的股票,
   買點提示會明白寫「現價追高,回檔守穩支撐再進較穩」。**評級看值不值得擁有,買點提示看該在什麼價位進場**,兩段一起讀。
2. **排序上強勢買進(3) 高於強烈推薦(2),但不代表報酬一定更高**,只是系統對該 setup 的信心層級。
   一個給順勢派、一個給價值派。

## 7. 怎麼讀個股分析輸出

一份個股報告會有:

- **綜合評分 + 評級**:四維度加權後的總分,與最終評級。
- **分項得分**:技術 / 動能 / 籌碼 / 估值 / 基本面(0~100)。看是哪個維度撐起或拖累總分。
- **基本面評價 / 動態趨勢 / 估值狀態 / 獲利品質**:白話標籤。
- **系統建議 + 買點提示**:含「大戶成本區 / 支撐 / 壓力」與「動態防守參考(停損)」。
  現價遠高於成本區時會標「早期累積區(僅供參考)」,並改以「現價下方最近高量支撐」為可操作成本區。
- **月營收即時動能註記**:季報有約 45 天公告時差,當月營收動能明顯轉強時,會提示「以月營收為即時依據」。

## 8. 綜合分選股怎麼用(海選整份名單)

這頁跟「個股分析 / 多檔排行」最大的差別:那兩頁是**即時抓最新資料、一檔一檔算**;這頁是讀一份**事先算好的分數快照**,把**整份自選名單**一次拿來比高下、由高到低排名。所以它 **0 API、免 token、瞬間出結果**,最適合「先海選一輪、再挑幾檔回個股分析看即時細節」。

**操作三步驟**

1. 左側選好**策略模式**(三個模式各一套權重、各一套分數,換模式排名就會變)。
2. 設四個篩選條件:**綜合分下限**(預設=該模式門檻)、**資料信心下限**、**顯示檔數**、**限定評級**(不選=全部)。
3. 看排名表,可**下載 CSV**;想深入某幾檔,把代號貼回「個股分析」看即時細節。

**欄位怎麼看**

- **綜合分**:五維(基本面、估值、技術、動能、籌碼)加權後的總分;權重依模式不同。
- **百分位**:這檔綜合分在**這份名單裡**的橫斷面排名(0～100,越高代表在名單中越前面)。
- **五維分數 / 估值狀態 / 信心**:各面向細分與資料完整度(信心低=部分資料缺漏,分數參考性打折)。
- **基準日 (as_of)**:這批分數是「哪一天」算出來的 —— 它是**快照、不是當下即時價**。

**重要:資料是快照,不是即時。** 想要最新即時的單檔狀態,回「個股分析」用你的 token 重抓。

### 8-1. 自選名單由你控制(watchlist.txt)

綜合分選股比的「整份名單」= 專案根目錄的 **`watchlist.txt`**,**完全由你自己維護**:

- **一行一檔**,只放代號即可(例:`2330`);代號後可加名稱/註解(例:`2330   # 台積電`)。
- **空白行、或 `#` 開頭整行**會略過 —— 可拿來分群、寫筆記。
- **重複代號自動去重**;**刪掉或清空整個檔案**則自動回退內建的 45 檔分散化測試池。

改完名單後要讓排名反映新名單,需**重算分數**:

```
python build_cache.py --build-scores          # 重算三個模式的綜合分 (0 API,讀本機原始快取)
```

- **本機/自架**:重算後回網頁按分頁上的「↻ 重新載入快取」即可看到新名單。
- **雲端部署站 (Streamlit Cloud)**:線上的綜合分選股讀的是 repo 內已提交的 `cloud_cache/Scores` 快照。
  所以**光改 `watchlist.txt` 不會馬上反映到線上** —— 需在本機重算後,把快照 commit / push 上去
  (執行 `python deploy_scores.py --rebuild-scores`,或雙擊 `update_and_push.bat`),Streamlit 才會重部署成新名單。

> 一句話:名單是你控制的;線上版要「重算 + 上傳」後才會更新。

## 9. 回測(驗證這套有沒有效)

`tests/run_backtest.py`,建議都加 `--cache` 走本機快取省 API:

| 指令 | 用途 |
|---|---|
| `python tests\run_backtest.py --cache` | 標準回測:分桶報酬 + 權益曲線 + 買進級診斷 |
| `python tests\run_backtest.py --cache --attribution` | 分項因子歸因:五維度各自的排序貢獻 (Rank IC) |
| `python tests\run_backtest.py --cache --cycle` | 多市場週期:2022 空頭 vs 2023–2025 多頭的排序力 |
| `python tests\run_backtest.py --cache --validate` | 樣本外驗證 (絕對報酬 vs 市場中性) |

判讀重點:這套的核心價值是**排序力**(市場中性:高分前⅓ − 低分後⅓ 的報酬差為正),
以及**風險調整後贏大盤**(Calmar),不是絕對報酬狂勝 —— 它「會排序、不會擇時」。

## 10. 省 API:本機快取

資料會自動落地本機快取,查過的之後自動重用 (詳見 `docs/快取與選股_CACHE.md`)。要點:

- 平常照舊執行即可,查過的股票自動進快取。
- 一次把整池建滿:`python build_cache.py --full`;每天增量:`python build_cache.py`。
- 要抓當日最新價:個股加 `--refresh`、回測加 `--cache-refresh`。

## 11. 常見問題 / 注意事項

- **這是投資建議嗎?** 不是。它是篩選/研究輔助,決策與風險由你自負。
- **貼了 token 還是抓不到?** 確認整串有複製到、帳號 Email 已完成驗證;短時間查太多檔可能撞每小時上限,等一小時額度重置。
- **不想註冊 FinMind 可以用嗎?** 可以用「綜合分選股」—— 0 API、免 token,只是資料是上次建快取的基準日、不是即時。
- **顯示的價格跟券商不一樣?** 可能是還原股價 (配息調整);下單以券商報價為準。
- **看到強勢買進就能追?** 不行。先看買點提示,若寫「追高」就等回檔;並自設停損與部位大小。
- **為什麼很少出現買進評級?** 這是「沒好貨就空手」的特性,不是壞事;想放寬可用平衡模式或擴大股票池 (改 `watchlist.txt`)。
- **資料可能有幾天延遲?** 快取為省 API 預設 2 天內視為新鮮,要最新用 `--refresh`。

## 12. 指令速查

```
# 個股 (命令列)
python main.py 2330                     # 分析單檔 (預設 balanced)
python main.py 2330 2454 -m c           # 多檔排行 + 保守模式
python main.py 2330 --refresh           # 強制重抓最新

# 個股 (網頁)
streamlit run app.py

# 回測 (都建議加 --cache)
python tests\run_backtest.py --cache
python tests\run_backtest.py --cache --attribution
python tests\run_backtest.py --cache --cycle

# 快取 / 綜合分選股
python build_cache.py --full            # 首次建庫
python build_cache.py                   # 每日增量
python build_cache.py --build-scores    # 重算五維綜合分 (綜合分選股用)
python build_cache.py --screen          # 跨股選股示範 (DuckDB)

# 部署綜合分到雲端 (改完 watchlist 重算後上傳)
python deploy_scores.py --rebuild-scores
```
