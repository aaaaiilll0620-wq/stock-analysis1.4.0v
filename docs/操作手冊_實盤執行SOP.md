# 實盤執行 SOP — 三層架構怎麼落地成每天的動作

> 這份文件回答一個具體問題：**每天/每次看到的名單、燈號都在變，我到底該不該動手？**
> 目的是把「選股邏輯」「操作頻率」「風控曝險」三件事釘死成不會混淆的固定規則，
> 防止臨場因為「今天名單看起來不一樣」而做出沒被回測驗證過的操作。

---

## 0. 三層架構（先分清楚，這是全份文件的地基）

系統其實是三條互相獨立的軸，各自回答不同問題，**不能互相取代、也不能合併判斷**：

| 層 | 回答什麼問題 | 誰決定 | 多久變一次 | 對應程式/文件 |
|---|---|---|---|---|
| **A. 選股層** | 這批股票要挑誰？ | 雙確認邏輯（c2 訊號等） | 每次掃描都會變（每天） | 全市場掃描 / 綜合分選股 |
| **B. 節奏層** | 什麼時候才能真的換股？ | 季度再平衡（≈60 交易日） | 每季 1 次 | `scripts/portfolio_simulator_lab.py --plan` |
| **C. 曝險層** | 現在該壓幾成資金（0/33/67/100%）？ | 市場燈號（多軸階梯+對稱確認3天） | 不定期，訊號觸發才變 | `core/regime_exposure.py`、App「🚦 市場燈號」分頁 |

**看到「名單變了」或「燈號變了」的第一個動作永遠是先問：這是 A、B、還是 C 在變？** 只有 B 觸發時才換股，A 自己變動不觸發任何操作，C 觸發時只調整既有持股的資金比例、不換股。

---

## 1. 建倉（第一次進場）

1. 用當天（B 層再平衡日）雙確認名單，**整批**買進（不要分批試單、不要挑順眼的先買）。
2. 記錄建倉日、成交價、股數 —— 這是下一次再平衡日比對的基準。
3. 建倉當下同時記錄當天曝險層狀態（0/33/67/100%），部位大小依此比例配置。

---

## 2. 持有期間（兩次再平衡之間，這段最長，也最容易手癢）

**預設動作：什麼都不做。**

會在這段期間內誘惑你操作的三種雜訊，一律忽略：

- **A 層每天的名單變動** → 不理會。今天名單有新股、你手上某檔掉出當天排名，都不代表要換股（[[streak-arm-result]] 已證實在榜天數/新舊沒有預測力）。
- **個股股價漲跌** → 不設個股停損、不因單一檔大跌就賣。整批操作優於逐股微調，已被回測證偽（batch2/3 部分調整 ΔM 全滅）。
- **燈號原始值單日翻動（`raw_above` 翻但 `pending`=True）** → 這只是還在 3 天確認中，不是訊號，等確認完成才算數（見第 3 節）。

這段期間唯一可能觸發操作的是 **C 層曝險訊號完成 3 天確認**（見下一節）。

---

## 3. C 層觸發：曝險變動時該做什麼（核心 SOP，防止「順手換股」）

### 3-1 降曝險（例：100% → 67%）

1. 確認 App「🚦 市場燈號」顯示的是**已確認狀態**（`pending=False`），不是原始單日翻動。
2. 把**現有持股**依比例減碼（例如 8 檔等權，降到 67% 曝險＝賣掉約 1/3 部位，8 檔可以是等比例減碼、或先賣 2~3 檔留現金，兩種等效即可）。
3. **不查看當天排行榜**、不做任何選股判斷——這一步純粹是「留多少錢在場內」，跟哪一檔股票好壞無關。

### 3-2 加回曝險（例：67% → 100%，這是你問到的情境）

1. 同樣先確認是**連續 3 天確認**後的狀態，不是原始單日翻動就衝進去。
2. **買回的對象＝原本持有、之前被減碼賣掉的同一批股票**，依比例補滿回原始部位。
3. **當天排行榜跟你原本持股不同的部分，一律不買**，不論看起來多誘人。理由：
   - 曝險回補的動作定義是「補回原班人馬」，不是「重新選股」。
   - 換股權只保留給 B 層（季度再平衡），這裡買新名字＝在兩次再平衡之間額外做了一次沒排定的部分換股，這正是回測證偽會侵蝕報酬的操作（[[portfolio-layer-strategy]] Q3：batch2/3 部分調整 ΔM −0.4~−0.6% 全滅）。
   - 排行榜上新出現的名字，記下來、留到下一次 B 層再平衡日才處理。

### 3-3 一句話版本

> **C 層只管「原班人馬要不要在場內」，不管「該不該換人」。換人只在 B 層發生。**

---

## 4. B 層觸發：季度再平衡日該做什麼

到了再平衡日（建倉日起約 60 個交易日後），才是唯一允許換股的時間點：

1. 重新跑一次雙確認名單（用當天的排行榜）。
2. **還留在榜上（如 Top-8）的持股** → 零成本續抱，不動作。
3. **掉出榜的持股** → 隔日開盤賣出，換成新入榜的股票（整批處理，不要今天賣一檔明天賣一檔）。
4. 換股後，回到第 2 節「持有期間」狀態，直到下一次再平衡日。

> 現況：機械化產出換股清單的工具（`--plan-exit`）尚未落地（見 `docs/工單_活體演練保護_第一梯隊.md` WP1-7），
> 目前需人工對照「當天排行榜 vs 手上持股」判斷 keep/sell，判斷規則就是本節第 2、3 點。

## 5. 常見誤區 Q&A（對照這份對話實際問過的情境）

| 情境 | 正確處理 | 為什麼不能做別的 |
|---|---|---|
| 今天名單跟昨天不一樣，要跟著換嗎？ | 不換，A 層每天變是正常雜訊 | streak 對報酬無預測力 |
| 買入後要抱 20 天還是設停損賣出？ | 都不是，抱到下次季度再平衡（≈60交易日） | 20 天是研究用前瞻窗，非持有規則；個股停損未被驗證 |
| 6/1 買入，8/1 回來調整？ | 不對，60 交易日≈3個月，應該約 9/1 才是再平衡日 | 8/1 只過約 2 個月，還沒到週期 |
| 燈號 100%→67%，我減碼了，之後又回 100%，怎麼辦？ | 買回原本持股，不看新排行榜 | C 層只管曝險比例，換股權在 B 層 |
| 曝險回補當天，排行榜有些不一樣，可以買這些不一樣的嗎？ | 不行，只買回原班人馬 | 提前換股＝未排定的部分調整，已證偽會侵蝕報酬 |
