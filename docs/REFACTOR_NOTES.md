# 台股 PIT 回測系統 — 順勢動能重構筆記

修改日期：2026-07-09　傾斜設定：**中性偏積極**　交付：寫回磁碟並保留 `.bak` 備份

## 根因：為什麼會變成「反向指標」

舊 `InvestmentAdvisor._decide_rating` 的【強烈推薦】要**同時**滿足：

- `valuation_ok` = 估值「偏低 / 合理」
- `dynamic_ok` = `total ≥ min_score` **且** `rsi < rsi_extreme` **且** `ma20_bias ≤ bias_chase`

主流飆股（2330 / 3661）漲上去後，天生就是**估值偏高、RSI 過熱、乖離大** → 同時卡死
`valuation_ok` 與 `dynamic_ok` → 永遠只能停在【觀望追蹤】。反之落底的弱勢股（2408）便宜、
RSI 低、乖離負 → 反而過關。這就是把「恐高 + 逆勢」寫進了關卡本身。

## 四大修改

### 1. 評級四級化 + 新增【強勢買進】順勢動能軌道（`advisor.py`）
- 新增 `RATING_SUPER = "強勢買進"`，排序 `{謹慎避開:0, 觀望追蹤:1, 強烈推薦:2, 強勢買進:3}`。
- `_super_buy_qualifies()`：**多頭排列 + 動能分≥mom_hot + 籌碼點火(whale熱且法人/主力實質流入)
  + 基本面未壞 + 非昂貴泡泡 + 未見衰竭出貨** → **特赦**估值/RSI/乖離，直接列最高級。
- `_rebound_rescue()`：被判【謹慎避開】但**技術突破關鍵壓力 + 籌碼顯著流入** → 強制改列
  【觀望追蹤】，修正南亞科型「落底反彈被完全錯過」的盲點。
- 【強烈推薦】維持原本「低估值價值股」定義不變。
- `_generate_advice` 新增強勢買進專屬語氣；`_trend_label` 新增「主流強勢(順勢動能)」。

### 2. 動態出場成為核心（`backtest.py`）
- `Backtester.run(exit_mode=...)` 預設由 `"horizon"` 改為 **`"dynamic_stop"`**。
- 進場後逐根 K 更新量價成本區：收盤實質跌破 POC/Support 或觸發 σ 移動停利(Chandelier)
  即果斷出場，不再固定死抱 20 天（合成回測實測平均持有 ~11 天、三種出場理由都有觸發）。

### 3. 核心五維度權重重配 + 動態權重（`scoring_manager.py` + `advisor.py`）
- `MODES.composite_weights` 全面調高 技術/動能/籌碼、壓低 估值/基本面（balanced：動能 0.15→0.26、
  籌碼 0.20→0.22、技術 0.20→0.22、估值 0.20→0.12、基本面 0.25→0.18）。
- **動態權重** `advisor._dynamic_weights()`：當個股「多頭排列 + 乖離≥8 + RSI≥55」時，
  自動把估值權重砍 60% 轉給 動能(0.6)/籌碼(0.4)，讓分數由順勢動能主導。

### 4. 回測統計全面支援四級（`backtest.py` / `tests/run_backtest.py`）
- `summarize / equity_curve / track_rating_changes / _score_cache_with / _bucket_spread / diagnose`
  全部改為四級；**多空價差 = 強勢買進平均報酬 − 謹慎避開平均報酬**（逐級退回）。
- `_score_one` 新增動能軌道診斷關卡：`g_多頭排列 / g_動能夠強 / g_籌碼點火 / g_非衰竭出貨`。
- `run_backtest.py` 新增 `--exit-mode`（預設 dynamic_stop）；`--only-strong` 改為只買 強勢買進+強烈推薦。

## 驗證
- 原 `test_advisor.py` 全部 8 個 `_decide_rating` + 5 個 composite 權重斷言**全數維持綠燈**
  （既有測試資料籌碼皆為 0，強勢買進與動態權重不會誤觸發）。
- 新增順勢動能 / 逆勢救援 / 泡沫剔除 / 衰竭保護共 12 條斷言全過。
- `Backtester.self_test()` 端到端跑通 dynamic_stop 與四級 summarize。
- 主流飆股端到端：RSI 79 / 乖離 22% / 估值偏高 → 正確判【強勢買進】、動態權重啟用、總分 80.2。

## v2 修正（依實測「慘狀」再大改）

**實測病灶**：主流飆股（2330/3661/3017 奇鋐/1519 華城）全部 dominant【觀望追蹤】，
2408 南亞科 +4.67% 仍是【謹慎避開】；權益曲線 +81.7% vs 0050 +119.9%（落後 38%）。

**根因**：v1 的【強勢買進】要求個股「已經極端」（RSI≥72 或 乖離>15 或 估值偏高）才特赦。
但**強勢但未極端**的主流股（RSI 60–68、乖離 8–14、估值不便宜）同時卡死「強勢買進(未極端)」
與「強烈推薦(不夠便宜)」兩道門 → 全部掉進【觀望追蹤】。這就是贏家全卡觀望的主因。

**v2 四項修正**：

1. **【強勢買進】拿掉「必須已極端」的要求** — 改用**穩健趨勢結構**（短均線多頭＋站穩週線＋
   中期未死叉）＋（價量或**營收**動能）＋**主力法人實質流入**認定；只要是有籌碼、有動能的
   主流強勢股就納入最高級，並特赦估值/RSI/乖離。用趨勢結構取代單日動能，避免月末快照被雜訊洗掉。

2. **【強烈推薦】新增「順勢型」通道** — 除原本價值型（便宜＋不過熱）外，
   新增「穩健多頭＋籌碼實質流入＋非泡沫/衰竭」通道，把**強但未極端**的主流股救出觀望桶。
   建議語另給順勢版本（不再對趨勢股喊「等回檔」）。

3. **門檻下修並引入營收動能** — mom_hot 58→50、whale_hot 52→45（balanced）；
   `_momentum_hot` 改為「價量動能 **或** 營收年增≥12%/連3月成長」任一即可。
   籌碼確認採兩級:強勢買進需 whale≥whale_hot、順勢強推需 whale≥chip_min，皆須有實質流入。

4. **逆勢救援放寬＋權益曲線集中化** — 救援改為「趨勢翻多(站上月線且5日線翻揚) 或 突破 + 籌碼流入」
   → 把南亞科型反彈股移出避開桶（改善反向指標）；`equity_curve` 預設由「強推+觀望」
   收斂為**只買 強勢買進+強烈推薦**，不再 closet-index 大盤（`--with-watch` 可還原較分散版本）。

5. **`summarize` 每次都印兩軌關卡未過率診斷** — 方便你逐次調門檻。

**驗證**：原 20 條斷言全綠；新增梯度案例：主流強勢+熱籌碼→強勢買進、+中籌碼→強烈推薦(順勢)、
無籌碼流入→觀望（不盲追）——三者在舊邏輯下全是觀望。

> ⚠️ **關於落後大盤**:0050 由台積電權重主導、又逢半導體大多頭;等權分散 36 檔(含金融/鋼鐵/面板落後股)
> 結構上很難贏過台積電權值。系統真正價值在「鑑別力(多空價差)」與集中主流。若要贏大盤,
> 方向是**依分數加權/集中強勢買進**而非等權買一籃 —— 已把預設權益曲線改為只買買進級。

## v3 依實測 432 筆關卡診斷精準調校

**實測瓶頸（強勢買進軌道「只差一道」58 筆）**：籌碼點火 30、動能夠強 16；
價值軌道「只差一道」34 筆：分數達門檻 17。據此做四項對症下藥：

1. **修正致命交互 bug（最關鍵）** — 舊順序把「估值偏高 + 綜合分<50」的軟避開排在強勢買進判定**之前**。
   主流飆股綜合分常被估值拖低到 50 以下 → 還沒輪到強勢買進就被錯殺成避開。已改為：
   **硬避開（基本面/現金流/破線）→ 強勢買進 → 估值型軟避開**。飆股不再因為「貴又分數低」被誤殺。

2. **動能改「三選一」** — `momentum_score≥mom_hot` **或** `technical_score≥tech_hot(65)` **或** 營收動能。
   技術結構分（多頭排列·MACD·週線）比單日動能穩定，讓結構強但當天沒大漲的leader也算動能達標。

3. **籌碼點火加「土洋同買 / 法人主導」替代路徑** — 不再只看 whale_score 數值（該分數天生偏低）。
   只要有實質流入且（whale≥whale_hot **或 投信+外資同步連買 或 法人成交占比≥40%**）即算點火。

4. **門檻下修** — whale_hot 45→42、mom_hot 50→46；min_score balanced 62→58、cons 68→64、aggr 58→55
   （對應「分數達門檻」17 筆一步之差）。

**驗證**：原 20 條斷言全綠;新梯度案例：結構強+土洋同買(單日動能低)→強勢買進、
土洋同買+whale低→強勢買進、僅單一外資流入+whale低→觀望(不盲追)。`_score_one` 診斷關卡已與實際判定式對齊。

> 下一輪再貼 `summarize` 的關卡表,若「籌碼點火/動能夠強」未過率明顯下降、強勢買進樣本變多即為正解;
> 若仍偏低,主要瓶頸可能在資料面（法人買賣/週線 MA 是否有正確灌進 PIT StockData），那要往 data_provider 追。
